# 双指针



## 移动零（283）

给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的**相对顺序**。

**请注意** ，必须在不复制数组的情况下原地对数组进行操作。

 **示例 1:**

```
输入: nums = [0,1,0,3,12]
输出: [1,3,12,0,0]
```

**示例 2:**

```
输入: nums = [0]
输出: [0]
```

```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) { // 传入vector<int>的引用，原地修改数组
        // 1. 初始化变量
        int n = nums.size(), left = 0, right = 0;
        // 2. 快指针遍历整个数组
        while (right < n) {
            // 3. 快指针找到非0元素时
            if (nums[right]) { // 等价于nums[right] != 0，非0值在bool判断中为true
                swap(nums[left], nums[right]); // 交换快慢指针位置的元素
                left++; // 慢指针后移，指向新的非0元素空位
            }
            right++; // 快指针始终后移，遍历下一个元素
        }
    }
};
```

**<span style="font-size:1.4em;">总结</span>**

这段代码是移动零问题的**最优双指针解法**，核心要点可概括为 3 点：

1. **快指针探路**：遍历数组找非 0 元素，无条件后移；
2. **慢指针占位**：指向非 0 元素的下一个放置位置，找到非 0 元素后才后移；
3. **交换赋值**：将非 0 元素放到慢指针位置，遍历结束后慢指针后全是 0，且非 0 元素相对顺序不变。

**复杂度分析**

- 时间复杂度：*O*(*n*)，其中 *n* 为序列长度。每个位置至多被遍历两次。
- 空间复杂度：*O*(1)。只需要常数的空间存放若干变量。



## 盛最多水的容器（11）

给定一个长度为 `n` 的整数数组 `height` 。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])` 。

找出其中的两条线，使得它们与 `x` 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

**说明：**你不能倾斜容器。

**示例 1：**

![img](https://aliyun-lc-upload.oss-cn-hangzhou.aliyuncs.com/aliyun-lc-upload/uploads/2018/07/25/question_11.jpg)

```
输入：[1,8,6,2,5,4,8,3,7]
输出：49 
解释：图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。在此情况下，容器能够容纳水（表示为蓝色部分）的最大值为 49。
```

**示例 2：**

```
输入：height = [1,1]
输出：1
```

```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        // 1. 初始化双指针和最大面积
        int l = 0, r = height.size() - 1;
        int ans = 0;
        // 2. 双指针向中间收缩，直到相遇
        while (l < r) {
            // 3. 计算当前指针围成的面积（核心公式）
            int area = min(height[l], height[r]) * (r - l);
            // 4. 更新全局最大面积
            ans = max(ans, area);
            // 5. 贪心移动指针：移较矮的那个
            if (height[l] <= height[r]) {
                ++l; // 左指针右移
            } else {
                --r; // 右指针左移
            }
        }
        // 6. 返回最大面积
        return ans;
    }
};
```

<span style="font-size:1.5em;">可推广到很多**区间最值 / 区间筛选**的数组问题中，是刷题必须掌握的经典解法</span>

**复杂度分析**

- 时间复杂度：*O*(*N*)，双指针总计最多遍历整个数组一次。
- 空间复杂度：*O*(1)，只需要额外的常数级别的空间。原地操作

## 三数之和（15）

给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意：**答案中不可以包含重复的三元组。

**示例 1：**

```
输入：nums = [-1,0,1,2,-1,-4]
输出：[[-1,-1,2],[-1,0,1]]
解释：
nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0 。
nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0 。
nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0 。
不同的三元组是 [-1,0,1] 和 [-1,-1,2] 。
注意，输出的顺序和三元组的顺序并不重要。
```

**示例 2：**

```
输入：nums = [0,1,1]
输出：[]
解释：唯一可能的三元组和不为 0 。
```

**示例 3：**

```
输入：nums = [0,0,0]
输出：[[0,0,0]]
解释：唯一可能的三元组和为 0 。
```

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        int n = nums.size();
        // 1. 对数组升序排序，为双指针和去重做准备
        sort(nums.begin(), nums.end());
        // 2. 定义结果集，存储所有不重复的三元组
        vector<vector<int>> ans;

        // 3. 枚举第一个数a，first是a的指针
        for (int first = 0; first < n; ++first) {
            // 去重a：如果当前a和上一个a相同，跳过（避免重复三元组）
            if (first > 0 && nums[first] == nums[first - 1]) {
                continue;
            }
            // 4. 双指针初始化：third指向数组末尾，target = -a（b+c需要等于target）
            int third = n - 1;
            int target = -nums[first];
            // 5. 枚举第二个数b，second是b的指针，从first+1开始（b在a的右侧）
            for (int second = first + 1; second < n; ++second) {
                // 去重b：如果当前b和上一个b相同，跳过（避免重复三元组）
                if (second > first + 1 && nums[second] == nums[second - 1]) {
                    continue;
                }
                // 6. 双指针核心：移动third，让b+c <= target（保证second < third）
                while (second < third && nums[second] + nums[third] > target) {
                    --third;
                }
                // 7. 终止条件：b和c指针重合，后续b增大，无满足条件的c，跳出内层循环
                if (second == third) {
                    break;
                }
                // 8. 找到有效三元组：b+c == target，即a+b+c=0，加入结果集
                if (nums[second] + nums[third] == target) {
                    ans.push_back({nums[first], nums[second], nums[third]});
                }
            }
        }
        // 9. 返回所有不重复的三元组
        return ans;
    }
};
```

复杂度分析

* 时间复杂度：O(N^2^)，其中 N 是数组 nums 的长度。

* 空间复杂度：O(logN)。我们忽略存储答案的空间，额外的排序的空间复杂度为 O(logN)。然而我们修改了输入的数组 nums，在实际情况下不一定允许，因此也可以看成使用了一个额外的数组存储了 nums 的副本并进行排序，空间复杂度为 O(N)。

## 接雨水（42）

给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

**示例 1：**

![img](https://assets.leetcode.cn/aliyun-lc-upload/uploads/2018/10/22/rainwatertrap.png)

```
输入：height = [0,1,0,2,1,0,1,3,2,1,2,1]
输出：6
解释：上面是由数组 [0,1,0,2,1,0,1,3,2,1,2,1] 表示的高度图，在这种情况下，可以接 6 个单位的雨水（蓝色部分表示雨水）。 
```

**示例 2：**

```
输入：height = [4,2,0,3,2,5]
输出：9
```



### 动态规划

该解法通过**预处理两个辅助数组**提前记录每个位置的左右最大高度，将暴力法的 O(n2) 时间复杂度优化到 O(n)，空间复杂度为 O(n)

**动态规划的优化思路**是：**提前遍历数组，用两个辅助数组预处理出每个位置的左侧最大高度、右侧最大高度**，后续直接查表计算，无需重复遍历，具体步骤：

1. **处理边界**：若数组为空，直接返回 0（无柱子无法接水）；
2. **构建 leftMax 数组**：`leftMax[i]` 表示**位置 i 及左侧所有柱子的最大高度**，从左到右遍历数组构建；
3. **构建 rightMax 数组**：`rightMax[i]` 表示**位置 i 及右侧所有柱子的最大高度**，从右到左遍历数组构建；
4. **累加总接水量**：遍历每个位置 i，用核心公式计算单位置接水量，累加所有结果得到总接水量。

```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        // 边界处理：空数组直接返回0
        if (n == 0) {
            return 0;
        }
        // 1. 构建leftMax数组：i及左侧的最大高度
        vector<int> leftMax(n);
        leftMax[0] = height[0]; // 第一个位置的左侧最大就是自己
        for (int i = 1; i < n; ++i) {
            // 第i个的左侧最大 = 前一个的左侧最大 和 自身高度的较大值
            leftMax[i] = max(leftMax[i - 1], height[i]);
        }

        // 2. 构建rightMax数组：i及右侧的最大高度
        vector<int> rightMax(n);
        rightMax[n - 1] = height[n - 1]; // 最后一个位置的右侧最大就是自己
        for (int i = n - 2; i >= 0; --i) {
            // 第i个的右侧最大 = 后一个的右侧最大 和 自身高度的较大值
            rightMax[i] = max(rightMax[i + 1], height[i]);
        }

        // 3. 累加所有位置的接水量，得到总结果
        int ans = 0;
        for (int i = 0; i < n; ++i) {
            // 核心公式：min(左最大, 右最大) - 自身高度
            ans += min(leftMax[i], rightMax[i]) - height[i];
        }
        return ans;
    }
};
```

复杂度分析

* 时间复杂度：O(n)，代码中包含三次独立的数组遍历（构建 leftMax、构建 rightMax、累加接水量），每次遍历的时间都是O(n)，无嵌套循环，因此整体时间复杂度为O(n)，是线性时间的最优解。

* 空间复杂度：O(n)，额外开辟了两个与原数组等长的辅助数组`leftMax`和`rightMax`，空间复杂度为O(n)；其余仅定义了几个整型变量，空间可忽略。

这段代码是接雨水问题的**经典动态规划解法**，也是理解该题所有最优解的基础，核心思路是**用空间换时间**，通过预处理辅助数组避免重复计算，核心要点可概括为 4 个：

1. **核心公式**：单位置接水量 = min (左侧最大高度，右侧最大高度) - 自身高度（木桶原理）；
2. **预处理数组**：左→右构建`leftMax`（i 及左侧最大），右→左构建`rightMax`（i 及右侧最大），状态转移方程为取前 / 后最大值与自身的较大值；
3. **无冗余判断**：辅助数组包含自身高度，保证计算结果非负，直接累加即可；
4. **复杂度最优**：时间O(n)（三次线性遍历），空间O(n)（两个辅助数组），是基础最优解。



### 单调栈

采用**单调递减栈**跟踪柱子索引，**一次遍历**即可找到每个**凹槽**的左右边界，直接计算凹槽的接水量并累加，无需预处理辅助数组，时间复杂度O(n)、空间复杂度O(n)（栈的开销

**从 “凹槽” 视角计算接水量**（动态规划是从 “单个位置” 视角），通过单调栈精准定位**凹槽的左边界、底部、右边界**，再利用矩形面积公式计算凹槽的接水量，是**单调栈解决数组凹槽 / 柱状图问题**的经典应用。

```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int ans = 0;                // 总接水量，初始为0
        stack<int> stk;             // 单调递减栈，存储【柱子的索引】（核心！不是高度）
        int n = height.size();      // 数组长度
        // 从左到右遍历每个柱子的索引i
        for (int i = 0; i < n; ++i) {
            // 核心循环：触发凹槽计算的条件（栈非空 + 当前高度>栈顶高度，找到右边界）
            while (!stk.empty() && height[i] > height[stk.top()]) {
                int top = stk.top();// 记录栈顶索引（凹槽的底部）
                stk.pop();          // 弹出底部，暴露新的栈顶（凹槽的左边界）
                if (stk.empty()) {  // 无左边界，无法形成凹槽，终止本次计算
                    break;
                }
                int left = stk.top();// 新的栈顶索引 = 凹槽的左边界
                int currWidth = i - left - 1;// 计算凹槽的宽度
                int currHeight = min(height[left], height[i]) - height[top];// 计算凹槽的有效接水高度
                ans += currWidth * currHeight;// 累加凹槽的接水量
            }
            stk.push(i);            // 保持栈单调递减：当前索引入栈
        }
        return ans;                 // 返回总接水量
    }
};
```

代码中栈`stk`存储的是**柱子的索引**（而非高度），且栈内索引对应的柱子高度严格满足**单调递减**，这个栈的唯一作用就是：**在遍历数组时，自动跟踪每个凹槽的底部和左边界，当遇到更高的柱子时，确定凹槽的右边界，进而计算凹槽接水量**。

单调递减栈的**核心规则**：

- 遍历到的柱子高度 **≤** 栈顶索引的柱子高度：直接将当前索引入栈，保持栈的单调递减；
- 遍历到的柱子高度 **>**栈顶索引的柱子高度：说明找到了**某个凹槽的右边界 **，此时触发栈的弹出操作，开始计算凹槽接水量。

**关键结论**：**每个柱子最多入栈一次、出栈一次**，因此遍历的实际时间复杂度为O(n)。

这段代码是接雨水问题的**经典单调栈最优解**，也是单调栈解决数组凹槽问题的典范，核心要点可概括为 4 个，记住这 4 点就能掌握该解法：

1. **栈存索引**：栈中存储柱子的**索引**（而非高度），用于计算凹槽宽度，同时可通过索引获取高度；
2. **栈的性质**：保持**单调递减栈**，目的是跟踪凹槽的底部和左边界，遇到更高的柱子时确定右边界；
3. **凹槽三要素**：触发 while 循环时，**当前 i = 右边界**，**弹出的栈顶 = 底部**，**新栈顶 = 左边界**（栈非空时）；
4. **核心公式**：凹槽接水量 = (右边界索引 - 左边界索引 - 1) × (min (左边界高度，右边界高度) - 底部高度)。

复杂度分析

* 时间复杂度：O(n)，其中 n 是数组 height 的长度。从 0 到 n?1 的每个下标最多只会入栈和出栈各一次。

* 空间复杂度：O(n)，其中 n 是数组 height 的长度。空间复杂度主要取决于栈空间，栈的大小不会超过 n。



### 双指针

```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int ans = 0; // 总接水量，初始为0
        // 初始化双指针：左指针在头，右指针在尾
        int left = 0, right = height.size() - 1;
        // 初始化两侧遍历到的最大高度，初始为0（无柱子时高度为0）
        int leftMax = 0, rightMax = 0;

        // 双指针向中间收缩，直到相遇/交叉
        while (left < right) {
            // 第一步：更新两侧的最大高度（先更新，再计算接水量，核心！）
            leftMax = max(leftMax, height[left]);
            rightMax = max(rightMax, height[right]);

            // 贪心处理短板侧：左矮处理左，右矮/相等处理右
            if (height[left] < height[right]) {
                // 左是短板，接水量=leftMax - height[left]
                ans += leftMax - height[left];
                ++left; // 左指针右移，处理下一个位置
            } else {
                // 右是短板，接水量=rightMax - height[right]
                ans += rightMax - height[right];
                --right; // 右指针左移，处理下一个位置
            }
        }
        return ans; // 返回总接水量
    }
};
```

`leftMax/rightMax`的定义是**当前指针位置及一侧的最大高度**，必须先将当前指针的高度纳入比较，才能得到正确的最大高度，否则会用 “未包含当前位置的旧 max” 计算，导致接水量为负（错误）。

这段代码是接雨水问题的**终极最优解**，将**木桶原理、贪心策略、双指针**完美结合，实现了时间和空间的双最优，核心要点可概括为 4 个，记住这 4 点就能彻底掌握：

1. **核心贪心**：比较当前左右指针的柱子高度，**短板侧的接水量由该侧遍历到的最大高度决定**（无需知道另一侧完整最大高度）；
2. **先更后算**：每次循环**先更新 leftMax/rightMax**，再计算接水量，保证 max 包含当前指针位置，避免计算错误；
3. **单步处理**：每次循环仅处理**一个位置**（短板侧），指针移动一步，所有位置仅被处理一次；
4. **极致优化**：用**两个变量**替代动态规划的两个辅助数组，用**一次遍历**替代单调栈的栈操作，实现空间 O (1)。

该解法的**贪心双指针**思路也是数组区间最值问题的经典优化思路，掌握后可应用到其他需要**预处理区间最值**的数组问题中，实现空间复杂度的极致优化。

* #### 时间 & 空间复杂度分析（终极最优）

  - 时间复杂度：O (n)

    双指针从两端向中间收缩，每个位置仅被处理一次，循环执行次数最多为 n-1 次，所有操作都是 O (1) 的常数操作，因此时间复杂度为严格的线性 O(n)；

  - 空间复杂度：O (1)

    仅定义了ans、left、right、leftMax、rightMax五个整型变量，无任何额外的数组 / 栈 / 容器开销，是真正的原地计算，空间复杂度为常数级 O (1)。

  这是接雨水问题**时间和空间都最优**的解法，没有任何优化空间。

|  单调栈  |     单调递减栈找凹槽，累加凹槽接水量     |    O(n)    |    O(n)    | 凹槽区域 | 一次遍历，适合柱状图类问题通用  | 需额外 O (n) 空间，栈操作稍复杂 |

<span style="font-size:1.9em; font-weight:bold; font-style:italic; text-decoration:underline;">双指针概念</span>

算法里的「双指针」**并非 C++ 语法上的指针类型**，而是用**两个整型索引变量**表示 “指针”，指向数组 / 字符串的某个位置，核心是**通过两个索引的协同移动，实现一次遍历完成逻辑，替代嵌套循环优化时间复杂度**。

|     类型     |    别名    |           初始位置           |          移动方向           |                    核心适用场景                    |         你接触的例题         |
| :----------: | :--------: | :--------------------------: | :-------------------------: | :------------------------------------------------: | :--------------------------: |
| **左右指针** | 相向双指针 |   左指针 l=0，右指针 r=n-1   |    向中间收缩（l++/--r）    | 数组 / 字符串的**区间最值、对称、两数 / 三数之和** |  盛最多水、三数之和、接雨水  |
| **快慢指针** | 同向双指针 | 慢指针 slow=0，快指针 fast=0 | 同方向遍历（++slow/++fast） |  数组 / 链表的**去重、移除元素、找中点、环检测**   | 移除有序数组重复项、链表中点 |

```cpp
#include <vector>
using namespace std;
int main() {
    vector<int> nums = {1,2,3,4,5};
    int n = nums.size(); // 先获取长度，避免重复计算，边界判断
    
    // 1. 定义双指针：整型索引变量，按场景初始化
    int l = 0, r = n - 1; // 左右指针（两端向中间）
    // int slow = 0, fast = 0; // 快慢指针（同向）
    
    // 2. 循环条件：按双指针类型定，核心是「不越界」
    while (l < r) { // 左右指针：避免相遇/交叉（根据场景可改l<=r）
        // while (fast < n) { // 快慢指针：快指针遍历完数组即终止
            // 3. 业务逻辑：根据问题写核心代码
            // 4. 移动指针：按规则单步/多步移动，核心是「不回退」
            l++; // 左指针右移
            r--; // 右指针左移
            // slow++; fast++; // 同向移动
        // }
    }
    return 0;
}
```

* **指针移动规则**：双指针的优化核心是**两个指针都只向一个方向移动，永不回退**（左指针只右移、右指针只左移、快慢指针只右移），确保总遍历步数≤n，时间复杂度 O (n)；

* **循环条件**：避免**数组越界**（比如`r >=0`、`fast <n`），左右指针常用`l < r`（接雨水、盛最多水），三数之和中内层双指针也是`l < r`；

* **长度提前计算**：`int n = nums.size();`，避免在循环中多次调用`nums.size()`（虽然 vector 的 size 是 O (1)，但养成习惯）；

* **边界预处理**：若数组可能为空，先判断`if (n == 0) return ...;`（比如接雨水、三数之和的代码都有此逻辑）。

```cpp
#include <algorithm> // 排序、min/max的头文件
#include <vector>
using namespace std;
vector<int> nums = {3,1,2,5,4};
// 1. 排序：双指针的常见前置操作（三数之和、去重）
sort(nums.begin(), nums.end()); // 升序排序，O(nlogn)
// 2. 最值：计算区间最值（接雨水、盛最多水）
int m = max(nums[l], nums[r]);
int m = min(nums[l], nums[r]);
// 3. 交换：左右指针实现数组反转、元素交换
swap(nums[l], nums[r]);
```



### 双指针两大基础模板（必背，可直接复用）

所有双指针的复杂题都是这两个基础模板的**延伸 / 组合**，比如三数之和是「固定单值 + 左右指针模板」，接雨水是「左右指针模板 + 贪心更新最值」，先背会最简模板，再做拓展。

#### 模板 1：左右指针（两端向中间收缩）【最常用】

**核心思路**：从数组两端开始，根据**业务条件**决定移动左指针还是右指针，向中间收缩直到相遇，一次遍历完成逻辑。

```cpp
// 功能：反转数组，时间O(n)，空间O(1)
void reverseArray(vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return;
    int l = 0, r = n - 1; // 初始：两端
    while (l < r) { // 循环：不相遇
        swap(nums[l], nums[r]); // 业务逻辑：交换元素
        l++; // 左指针右移
        r--; // 右指针左移
    }
}
```

**适用场景拓展**：

- 盛最多水：业务条件是「移动较矮的指针」，交换改为「计算面积并更新最大值」；
- 接雨水：业务条件是「移动短板侧指针」，交换改为「更新最值并计算接水量」；
- 三数之和：固定 first 指针后，内层用此模板，业务条件是「根据三数之和大小移动指针」。

#### 模板 2：快慢指针（同向遍历）【基础必备】

**核心思路**：快指针**探路**（遍历数组，寻找符合条件的元素），慢指针**记录有效位置**（指向结果数组的最后一个有效元素），快指针遍历完后，慢指针的位置就是结果的长度。

```cpp
// 功能：移除有序数组的重复项，返回新长度，时间O(n)，空间O(1)
int removeDuplicates(vector<int>& nums) {
    int n = nums.size();
    if (n == 0) return 0;
    int slow = 0; // 慢指针：记录有效元素的最后位置
    for (int fast = 1; fast < n; fast++) { // 快指针：从1开始探路
        // 业务条件：快指针找到和慢指针不同的元素
        if (nums[fast] != nums[slow]) {
            slow++; // 慢指针移动到下一个有效位置
            nums[slow] = nums[fast]; // 覆盖重复元素，保留有效元素
        }
    }
    return slow + 1; // 慢指针是索引，长度+1
}
```

**什么时候该用双指针**：

1. **指针不回退**：两个指针都只向一个方向移动，总遍历步数**不超过 n**（左右指针总步数 n，快慢指针总步数 n），避免了嵌套循环的重复遍历；
2. **一次遍历完成多任务**：两个指针协同工作，快指针 / 右指针负责 “探路 / 遍历”，慢指针 / 左指针负责 “记录 / 计算”，一次遍历完成原本需要两次嵌套遍历的工作。

## 总结（双指针基础核心要点）

1. **算法双指针≠C++ 语法指针**：数组 / 字符串中就是**整型索引变量**，核心是索引的协同移动；
2. **两大基础模板**：**左右指针（两端向中间）** 处理区间 / 对称 / 求和问题，**快慢指针（同向）** 处理去重 / 移除元素问题，是所有双指针题的基础；
3. **核心优化逻辑**：指针**不回退**，总遍历步数 O (n)，替代嵌套循环，将时间复杂度从 O (n?) 优化到 O (n)；
4. **关键语法细节**：先预处理边界（空 / 长度不足）、指针只向一个方向移动、循环条件避免越界；
5. **常用前置操作**：需要调整和 / 大小关系的场景，**先对数组排序**（比如三数之和、两数之和 II）。
