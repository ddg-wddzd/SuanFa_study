## 和为K的子数组（560）

给你一个整数数组 `nums` 和一个整数 `k` ，请你统计并返回 *该数组中和为 `k` 的子数组的个数* 。

子数组是数组中元素的连续非空序列。

**示例 1：**

```
输入：nums = [1,1,1], k = 2
输出：2
```

**示例 2：**

```
输入：nums = [1,2,3], k = 3
输出：2
```

### 枚举

```cpp
class Solution {
public:
    // 函数功能：统计nums中「和为k的连续子数组」的个数
    // 形参：nums-输入整数数组，k-目标和；返回值：符合条件的子数组数量
    int subarraySum(vector<int>& nums, int k) {
        const int n = nums.size(); // 数组长度，用const修饰表示只读，避免误修改
        int count = 0; // 核心：统计和为k的子数组个数，初始化为0
        int arr[n];    // 定义数组拷贝nums（★ 冗余点：无必要拷贝，直接用nums即可）
        // 把nums的元素拷贝到arr中（★ 冗余点：完全可删除该循环，直接操作nums）
        for (int i = 0; i < n; i++) {
            arr[i] = nums[i];
        }
        // 枚举子数组的「结尾位置」start（本质：遍历每个可能的子数组结尾）
        for (int start = 0; start < nums.size(); ++start) {
            int sum = 0; // 临时变量：存储当前子数组的累加和，每次换结尾重置为0
            // 从结尾start倒着枚举「起始位置」end，子数组为[end, start]（连续）
            for (int end = start; end >= 0; --end) {
                sum += arr[end]; // 累加当前元素，更新子数组和
                if (sum == k) {  // 累加和等于目标k，符合条件
                    count++;     // 统计数+1
                }
            }
        }
        return count; // 返回最终统计结果
    }
};
```

复杂度分析

* 时间复杂度：O(n^2^)，其中 n 为数组的长度。枚举子数组开头和结尾需要 O(n^2^)的时间，其中求和需要 O(1) 的时间复杂度，因此总时间复杂度为 O(n^2^ )。

- 空间复杂度：*O*(1)。只需要常数空间存放若干变量。

### 前缀和 + 哈希表优化

```cpp
class Solution {
public:
    // 函数功能：统计nums中「和为k的连续子数组」的个数
    // 核心思路：前缀和 + 哈希表，O(n)时间/空间，暴力法的最优优化
    int subarraySum(vector<int>& nums, int k) {
        // 哈希表mp：键=前缀和pre，值=该前缀和出现的次数（核心设计）
        unordered_map<int, int> mp;
        mp[0] = 1; // ★ 最关键的初始化：前缀和为0的情况，出现1次（必背！）
        int count = 0; // 统计符合条件的子数组总数，初始0
        int pre = 0;   // 实时计算的「当前前缀和」，初始0（pre[0]=0的定义）
        // 单层遍历数组，逐个元素更新前缀和（遍历到第i个元素，pre就是pre[i+1]）
        for (auto& x : nums) { // x是nums当前遍历的元素，用&避免拷贝，提升效率
            pre += x; // 累加当前元素，更新为最新的前缀和
            // 核心判断：若哈希表中存在「pre - k」，说明存在子数组和为k
            // mp.find(key)：找到返回对应迭代器，没找到返回mp.end()
            if (mp.find(pre - k) != mp.end()) {
                // 累加「pre - k」的出现次数 → 这是「以当前元素为结尾」的符合条件的子数组个数
                count += mp[pre - k];
            }
            // 将「当前前缀和pre」加入哈希表：出现次数+1（若已存在则累加，不存在则设为1）
            mp[pre]++;
        }
        return count; // 返回最终统计结果
    }
};
```

复杂度分析

* 时间复杂度：O(n)，其中 n 为数组的长度。我们遍历数组的时间复杂度为 O(n)，中间利用哈希表查询删除的复杂度均为 O(1)，因此总时间复杂度为 O(n)。

* 空间复杂度：O(n)，其中 n 为数组的长度。哈希表在最坏情况下可能有 n 个不同的键值，因此需要 O(n) 的空间复杂度。



## 滑动窗口最大值(239)

给你一个整数数组 `nums`，有一个大小为 `k` 的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 `k` 个数字。滑动窗口每次只向右移动一位。

返回 *滑动窗口中的最大值* 。

 **示例 1：**

```
输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口的位置                最大值
---------------               -----
[1  3  -1] -3  5  3  6  7       3
 1 [3  -1  -3] 5  3  6  7       3
 1  3 [-1  -3  5] 3  6  7       5
 1  3  -1 [-3  5  3] 6  7       5
 1  3  -1  -3 [5  3  6] 7       6
 1  3  -1  -3  5 [3  6  7]      7
```

**示例 2：**

```
输入：nums = [1], k = 1
输出：[1]
```

### 优先队列

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        int n = nums.size();
        priority_queue<pair<int, int>> q; // 大顶堆：<值,索引>，按值降序
        // 初始化第一个窗口
        for(int i=0; i<k; i++) q.emplace(nums[i], i);
        vector<int> ans = {q.top().first};
        // 滑动窗口，懒删除无效堆顶
        for(int i=k; i<n; i++){
            q.emplace(nums[i], i);
            // 弹出窗口外的堆顶元素（有效窗口[i-k+1, i]）
            while(q.top().second <= i-k) q.pop();
            ans.push_back(q.top().first);
        }
        return ans;
    }
};
```

复杂度分析

* 时间复杂度：O(nlogn)，其中 n 是数组 nums 的长度。在最坏情况下，数组 nums 中的元素单调递增，那么最终优先队列中包含了所有元素，没有元素被移除。由于将一个元素放入优先队列的时间复杂度为 O(logn)，因此总时间复杂度为 O(nlogn)。

* 空间复杂度：O(n)，即为优先队列需要使用的空间。这里所有的空间复杂度分析都不考虑返回的答案需要的 O(n) 空间，只计算额外的空间使用。

### 单调队列

```cpp
class Solution {
public:
    // 函数功能：求大小为k的滑动窗口在nums中滑动时，每个窗口的最大值
    // 形参：nums-输入整数数组，k-固定滑动窗口大小；返回值：各窗口最大值组成的数组
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        int n = nums.size(); // 数组长度
        deque<int> q;        // 核心：双端队列，仅存储【nums的索引】，不存值！
                             // 队列特性：按索引对应的值【单调递减】排列，队头是当前窗口最大值索引
        // 第一步：初始化第一个窗口（前k个元素），构建初始单调递减队列
        for (int i = 0; i < k; ++i) {
            // 维护队列单调递减：若队非空，且当前元素 >= 队尾索引对应的值
            // 则弹出队尾（这些元素不可能成为后续任何窗口的最大值，直接淘汰）
            while (!q.empty() && nums[i] >= nums[q.back()]) {
                q.pop_back(); // 从队尾弹出（双端队列的尾部操作）
            }
            q.push_back(i); // 将当前元素索引加入队尾，保持队列单调递减
        }

        // 第一个窗口的最大值 = 队头索引对应的值，初始化结果数组
        vector<int> ans = {nums[q.front()]};

        // 第二步：滑动窗口，从第k个元素开始（索引k）遍历到数组末尾
        for (int i = k; i < n; ++i) {
            // 步骤1：维护队列单调递减（和初始化窗口的逻辑完全一致）
            // 新元素入队前，淘汰队尾所有比它小/相等的元素索引
            while (!q.empty() && nums[i] >= nums[q.back()]) {
                q.pop_back();
            }
            q.push_back(i); // 将当前元素索引加入队尾

            // 步骤2：过滤窗口外的无效队头（保证队头在当前窗口内）
            // 窗口有效索引范围：[i-k+1, i]，队头索引 <= i-k 表示已滑出窗口，弹出
            while (q.front() <= i - k) {
                q.pop_front(); // 从队头弹出（双端队列的头部操作）
            }

            // 步骤3：此时队头一定是当前窗口的最大值索引，加入结果数组
            ans.push_back(nums[q.front()]);
        }

        return ans; // 返回所有窗口的最大值
    }
};
```

复杂度分析

* 时间复杂度：O(n)，其中 n 是数组 nums 的长度。每一个下标恰好被放入队列一次，并且最多被弹出队列一次，因此时间复杂度为 O(n)。

* 空间复杂度：O(k)。与方法一不同的是，在方法二中我们使用的数据结构是双向的，因此「不断从队首弹出元素」保证了队列中最多不会有超过 k+1 个元素，因此队列使用的空间为 O(k)。



### 分块 + 预处理

```cpp
class Solution {
public:
    // 函数功能：求大小为k的滑动窗口在nums中滑动时，每个窗口的最大值
    // 核心思路：分块预处理前后缀最值，O(n)时间/空间，无嵌套循环，线性遍历实现
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        int n = nums.size();
        // 前缀最大值数组：prefixMax[i] = 第i个元素所在块中，从【块起始】到【i】的最大值（块内左→右的最值）
        vector<int> prefixMax(n);
        // 后缀最大值数组：suffixMax[i] = 第i个元素所在块中，从【i】到【块结束】的最大值（块内右→左的最值）
        vector<int> suffixMax(n);

        // 第一步：预处理prefixMax数组（从左到右遍历，按k分块）
        for (int i = 0; i < n; ++i) {
            if (i % k == 0) { // i是当前块的【起始位置】（每k个元素一个块）
                prefixMax[i] = nums[i]; // 块起始的前缀最值就是自己
            } else { // 非块起始，前缀最值=前一个位置的前缀最值 和 当前元素的较大值
                prefixMax[i] = max(prefixMax[i - 1], nums[i]);
            }
        }

        // 第二步：预处理suffixMax数组（从右到左遍历，按k分块）
        for (int i = n - 1; i >= 0; --i) {
            // i是当前块的【结束位置】：要么是数组最后一个元素，要么i+1是下一个块的起始（(i+1)%k==0）
            if (i == n - 1 || (i + 1) % k == 0) {
                suffixMax[i] = nums[i]; // 块结束的后缀最值就是自己
            } else { // 非块结束，后缀最值=后一个位置的后缀最值 和 当前元素的较大值
                suffixMax[i] = max(suffixMax[i + 1], nums[i]);
            }
        }

        // 第三步：遍历所有滑动窗口，计算每个窗口的最大值
        vector<int> ans;
        // 窗口起始索引i的范围：0 ~ n-k（保证窗口结束索引i+k-1 < n，不越界）
        for (int i = 0; i <= n - k; ++i) {
            int j = i + k - 1; // 窗口结束索引
            // 核心：当前窗口[i,j]的最大值 = max(窗口左端点i的后缀最值, 窗口右端点j的前缀最值)
            ans.push_back(max(suffixMax[i], prefixMax[j]));
        }

        return ans;
    }
};
```

复杂度分析

* 时间复杂度：O(n)，其中 n 是数组 nums 的长度。我们需要 O(n) 的时间预处理出数组 prefixMax，suffixMax 以及计算答案。

* 空间复杂度：O(n)，即为存储 prefixMax 和 suffixMax 需要的空间。

## 最小覆盖子串(76)

给定两个字符串 `s` 和 `t`，长度分别是 `m` 和 `n`，返回 s 中的 **最短窗口 子串**，使得该子串包含 `t` 中的每一个字符（**包括重复字符**）。如果没有这样的子串，返回空字符串 `""`。

测试用例保证答案唯一。

**示例 1：**

```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。
```

**示例 2：**

```
输入：s = "a", t = "a"
输出："a"
解释：整个字符串 s 是最小覆盖子串。
```

**示例 3:**

```
输入: s = "a", t = "aa"
输出: ""
解释: t 中两个字符 'a' 均应包含在 s 的子串中，
因此没有符合条件的子字符串，返回空字符串。
```

### 滑动窗口

```cpp
class Solution {
public:
    // 全局哈希表（也可定义在函数内，方便check函数访问）
    unordered_map<char, int> ori; // 存储【目标串t】的字符计数：key=字符，value=该字符在t中需要的最小数量
    unordered_map<char, int> cnt; // 存储【当前滑动窗口】的目标字符计数：key=字符，value=该字符在窗口中的数量

    // 核心校验函数：判断当前窗口是否【合法】（是否包含t的所有字符，且每个字符的数量都满足要求）
    bool check() {
        // 遍历目标串的字符需求ori，逐一验证窗口内的计数是否达标
        for (const auto &p : ori) {
            // 若窗口内某目标字符的数量 < 需求数量 → 窗口不合法，返回false
            if (cnt[p.first] < p.second) {
                return false;
            }
        }
        // 所有目标字符的计数都达标 → 窗口合法，返回true
        return true;
    }

    // 主函数：找s中包含t所有字符的最小覆盖子串
    string minWindow(string s, string t) {
        // 第一步：初始化目标字符需求表ori（统计t中每个字符的出现次数）
        for (const auto &c : t) {
            ++ori[c];
        }

        // 滑动窗口双指针：l=左指针（窗口起始），r=右指针（窗口结束），初始r=-1表示窗口初始为空
        int l = 0, r = -1;
        int len = INT_MAX; // 记录最小覆盖子串的长度，初始为整型最大值（方便后续更新）
        int ansL = -1;     // 记录最小覆盖子串的【起始索引】，初始-1表示未找到合法窗口
        // 无需ansR：因为知道起始索引和长度，可通过substr直接截取，也可记录ansR=r

        // 第二步：滑动窗口核心循环：右指针未遍历完s时，持续扩张/收缩窗口
        // 注意：s.size()是无符号整数，需转int避免r（int）和无符号数比较的类型问题
        while (r < int(s.size())) {
            // 1. 扩张右指针：将下一个字符纳入窗口，直到窗口合法
            // 先++r再取s[r]，避免初始r=-1的越界问题
            if (ori.find(s[++r]) != ori.end()) {
                // 仅当当前字符是【目标字符】（在ori中存在），才更新窗口计数cnt（非目标字符无需统计）
                ++cnt[s[r]];
            }

            // 2. 收缩左指针：当窗口合法时，尽可能左移l缩小窗口，找当前合法状态下的最小窗口
            // 循环条件：窗口合法 && 左指针不超过右指针（窗口非空）
            while (check() && l <= r) {
                // 若当前窗口长度 < 已记录的最小长度 → 更新最小窗口信息
                if (r - l + 1 < len) {
                    len = r - l + 1; // 更新最小长度
                    ansL = l;        // 更新最小窗口的起始索引
                }
                // 左移前：若左指针指向的是目标字符，需减少窗口计数cnt（因为该字符要移出窗口）
                if (ori.find(s[l]) != ori.end()) {
                    --cnt[s[l]];
                }
                ++l; // 左指针右移，收缩窗口
            }
        }

        // 最终判断：若ansL仍为-1，说明无合法窗口（s不包含t的所有字符），返回空串；否则截取子串
        // s.substr(起始索引, 子串长度)：核心是第二个参数是长度，不是结束索引！
        return ansL == -1 ? string() : s.substr(ansL, len);
    }
};
```

复杂度分析

* 时间复杂度：最坏情况下左右指针对 s 的每个元素各遍历一遍，哈希表中对 s 中的每个元素各插入、删除一次，对 t 中的元素各插入一次。每次检查是否可行会遍历整个 t 的哈希表，哈希表的大小与字符集的大小有关，设字符集大小为 C，则渐进时间复杂度为 O(C?∣s∣+∣t∣)。
* 空间复杂度：这里用了两张哈希表作为辅助空间，每张哈希表最多不会存放超过字符集大小的键值对，我们设字符集大小为 C ，则渐进空间复杂度为 O(C)。
